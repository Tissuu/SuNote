## 電影裡的 AI 或許會成真?! ----前言
大家應該都有看過威爾史密斯主演的機械公敵，或是阿諾史瓦辛格主演的魔鬼終結者。  
電影裡的故事常常描述著一個橫跨全世界的人工智慧，威脅人類文明的未來，最後展開全面戰爭。  
這類故事圍繞著一個很重要的核心問題，那就是機器到底有沒有自我意識，甚至會不會有感情。

不過回到現實世界，目前我們常用的 AI，像是 ChatGPT、Claude、Gemini 這類工具，其實還沒有到電影那種會自己產生意志、想統治世界的程度。  
它們比較像是一個吸收過大量資料，然後可以根據我們輸入的內容，快速產生文字、程式碼、摘要或建議的工具。

所以今天就先不談太科幻的部分，而是用比較白話的方式整理一下：AI 到底是什麼? 它又是怎麼回應我們的?

## AI 到底是什麼?
AI 是 Artificial Intelligence 的縮寫，中文通常翻成人工智慧。  
簡單來說，就是讓電腦去模擬一些原本看起來需要人類智慧才能完成的事情。

例如：

<ul>
  <li>看到圖片後判斷裡面是不是一隻貓</li>
  <li>讀完一段文章後整理成重點</li>
  <li>根據使用者的問題產生回答</li>
  <li>幫我們分類資料、預測結果或產生程式碼</li>
</ul>

但這裡要先釐清一件事：  
AI 會回答問題，不代表它真的「懂」這個世界；AI 可以產生很像人類寫出來的句子，也不代表它真的有情緒或自我意識。  
比較精準一點的說法是，它是透過大量資料與演算法，學會在某些情境下產生看起來合理的結果。

## 現在常見的 AI：LLM
現在大家最常接觸到的 AI，大多是 LLM，也就是 Large Language Model，大型語言模型。  
像 ChatGPT 這類工具，就是語言模型的應用之一。

LLM 的核心概念可以先簡化成一句話：  
它看過大量文字資料，然後根據前面的內容，預測下一段最可能出現的文字。

例如我們輸入：

今天天氣很好，我想去


模型可能會接著產生：

公園散步。


這不是因為它真的想去公園，而是因為在它學過的語言資料裡，「天氣很好」後面接「去公園」、「出去走走」、「曬太陽」這些內容的機率很高。  
所以它會根據上下文，選出一段看起來最合理的回答。

也因為如此，AI 有時候會講得很像真的，但內容不一定完全正確。  
這也是為什麼使用 AI 的時候，不能完全照單全收，尤其是遇到程式碼、法律、醫療、投資這種比較重要的內容，更需要自己再檢查一次。

## Token 是什麼?
在使用 AI 工具的時候，常常會看到一個詞叫 token。  
一開始看到真的會有點抽象，因為它不是單純等於一個中文字，也不一定等於一個英文單字。

可以先把 token 想成模型處理文字時的「小單位」。  
模型不是真的像人類一樣一整句一整句理解，而是會先把文字拆成許多 token，再拿這些 token 去做運算。

例如英文句子可能會被拆成單字、字根或符號；中文也可能會依照模型的規則被拆成不同片段。  
所以同樣一段文字，對人類來說可能只是短短一句話，但對模型來說可能已經是好幾個 token。

token 會影響幾件事：
<ul>
  <li>可以輸入多少內容</li>
  <li>AI 可以記住多長的上下文</li>
  <li>使用 API 時的計費方式</li>
  <li>模型產生回答時需要消耗多少資源</li>
</ul>

簡單講，如果把 AI 想成正在讀一篇文章的人，token 就像是它一次一次吃進去處理的文字碎片。

## Prompt 為什麼重要?
Prompt 就是我們給 AI 的指令或問題。  
很多時候 AI 回答得好不好，不只跟模型本身有關，也跟 prompt 寫得清不清楚有很大的關係。

例如如果只輸入：
```txt
幫我寫文章
```

AI 可能會不知道你要什麼主題、語氣、長度、對象，也不知道要不要附程式碼。  
但如果改成：
```txt
請用初學者看得懂的語氣，寫一篇介紹 AI、LLM、token 和 prompt 的技術文章，長度約 1000 字。
```

這樣 AI 就比較容易產生符合需求的內容。

所以 prompt 可以想成是在跟 AI 溝通任務需求。  
你給的條件越清楚，它就越知道要往哪個方向回答。  
這點其實跟工程師寫需求文件有點像，需求越模糊，最後做出來的東西就越容易歪掉。

## AI Agent 是什麼?
除了單純聊天的 AI 之外，現在也很常聽到 AI Agent。  
Agent 可以先理解成「比較會做事的 AI 助手」。

一般聊天機器人多半是你問一句，它回一句。  
但 Agent 會更進一步，它可能會根據一個目標去拆解任務，然後使用工具完成工作。

例如你給它一個任務：
```txt
幫我整理這個專案的 README，並檢查有沒有缺少安裝步驟。
```

比較完整的 Agent 可能會：
<ul>
  <li>先讀取專案檔案</li>
  <li>找出 package.json 或設定檔</li>
  <li>理解專案如何啟動</li>
  <li>整理 README 缺少的內容</li>
  <li>最後提出修改建議或直接協助修改</li>
</ul>

這就不只是單純產生文字，而是開始有一點「規劃任務、使用工具、完成目標」的感覺。  
不過即使是 Agent，也還是需要人類檢查。因為它可能誤判檔案內容、搞錯需求，或是產生看起來合理但其實有問題的結果。

## 小結
以前看電影時，AI 常常被描寫成一種會自我進化、產生意識，最後反過來控制人類的存在。  
但以目前常見的 AI 工具來說，它更像是一個超強的輔助工具，而不是電影裡那種完全自主的意識體。

它可以幫我們查方向、寫草稿、整理資料、產生程式碼，也可以在學習時當成一個很方便的陪練對象。  
但 AI 產生的內容不一定永遠正確，所以真正重要的還是使用者本身要知道自己想做什麼，也要有能力判斷答案能不能用。

簡單整理今天的重點：
<ul>
  <li>AI 是讓電腦模擬人類判斷、生成、分類與決策的技術</li>
  <li>LLM 是大型語言模型，會根據上下文預測合理的文字</li>
  <li>token 是模型處理文字的基本單位</li>
  <li>prompt 是我們給 AI 的任務說明</li>
  <li>AI Agent 可以拆任務和使用工具，但仍然需要人類檢查</li>
</ul>

感覺前幾天已經喝了不少雞湯，差不多也該飽了。今天就輕鬆一點，來閒話家常一下。  
台灣在 2025 年的交通事故死亡人數約為 2858 人，其中與機車相關的事故占比遠遠超過一半，高達驚人的 65%，也就是一年大約有近 1800 人因機車事故喪命。  
記得小時候，能接觸到的交通資訊來源，大多只有第四台新聞媒體；但隨著自媒體興起，近幾年關於台灣交通的討論也變得越來越多元。從道路設計、駕駛習慣，到行人地獄等議題，大家都有各自的觀察與想法。    
不過，好像比較少人會去討論「停車位」這件事。  
根據網路資料，雙北的機車總量加起來超過 300 萬輛，但台北與新北兩個縣市的機車停車格合計卻只有 80 多萬格。換句話說，平均每 3 個人，就得競爭 1 個車位。  
在供需如此失衡的情況下，找停車位有時候甚至比實際通勤還要辛苦。也讓人開始思考：既然大家都知道台灣交通環境不算友善，為什麼還是有這麼多人每天冒著風險騎車上路？  
或許答案其實很現實。  
不是每個地方的大眾運輸都足夠方便；而台灣的氣候、通勤距離與都市結構，也讓機車成為效率最高的移動工具之一。再加上機車的購車與養車成本相對低廉，對許多人來說，它並不是「想不想騎」的問題，而是「不得不騎」的選擇。  
很多交通問題，或許也不只是單純的用路人素質問題，而是整個城市空間與交通資源早已長期失衡。當道路、停車位與公共運輸跟不上人們的需求時，人們最後只能在有限的選擇裡，找到一種勉強能讓生活繼續運轉的方法。   
而會想到這些也只是因為我早上找了30分鐘的車位... 希望大家都可以順利找到自己理想的車位...

## git clone 複製專案到本地
<ul>
<li>下載整個專案</li>
<li>自動建立.git</li>
<li>自動設定origin</li>
</ul>

```git clone <url>``` 
## git fetch 從遠端下載最新變更

<ul>
<li>更新遠端分支(HEAD不會移動)</li>
<li>不會更動本地程式碼</li>
</ul>

```git fetch origin``` 抓取所有遠端變更  
```git fetch origin dev``` 只抓取dev分支  
```git fetch --all``` 抓取所有遠端分支  

## git pull 從遠端下載最新變更

<ul>
<li>更新遠端分支</li>
<li>更新目前所在的本地分之(HEAD會移動)</li>
<li>分叉時會產生commit emerge</li>
</ul>

## 利用GitHub內建功能開ISSUES
<ul>
<li>確認沒有重複的issue</li>
<li>標題簡短清楚，可以加點數分類</li>
<li>用 Label 標註性質,Milestone 管理階段目標</li>
<li>每人一次只接一張,assign 給自己</li>
<li>完成後關閉 issue(手動或 PR 加 close)</li>
</ul>

## pull request  
你寫好功能  
   ↓  
開 PR，告訴大家「我做好了」  
   ↓  
同事審查、留言討論  
   ↓  
修改完善  
   ↓  
核准 → 合併進 main  
<ul>
<strong>PR注意事項</strong>
<li>確認功能正確再發</li>
<li>描述清楚做了什麼</li>
<li>附圖或測試結果</li>
<li>異動不要太大</li>
<li>PR前先rebase</li>
</ul>

## git 暫存區
將當前的工作區變更，暫時儲存起來  
基本指令如下:  
```git stash``` 將當前的工作區變更暫時儲存起來    
```git stash list``` 查看暫存區內的變更  
```git stash -u```暫存修改包含未追蹤的檔案   
```git stash pop``` 套用最新stash 並刪除該stash  
```git stash pop @{n}``` 套用指定stash並刪除   
```git stash branch <分支名> stash@{n}``` 用指定stash建立分支並套用  

## git 摘取
指令如下:  
```git cherry-pick <commit sha>``` 從其他分支挑選指定的 commit 複製
到目前分支  
<strong>SHA值不同但內容一樣</strong>   
<strong>解衝突後 ```git cherry-pick --continue``` </strong>
